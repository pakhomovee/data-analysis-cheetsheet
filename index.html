Probability Essentials

Core formulas for tasks A-3 and B-5: conditional probability, total probability, Bayes' theorem, inclusion-exclusion, expectation and variance.

$P(A|B)=\dfrac{P(A \cap B)}{P(B)}$

$P(A \cap B)=P(A|B)P(B)=P(B|A)P(A)$

$P(A \cup B)=P(A)+P(B)-P(A \cap B)$

$P(A \cup B \cup C)=\sum P(A_i)-\sum P(A_i \cap A_j)+P(A \cap B \cap C)$

ConceptFormula / meaningExam cue
Independence$P(A \cap B)=P(A)P(B)$One event does not change probability of the other.
Mutually exclusive$P(A \cap B)=0$Cannot happen together. Not the same as independent.
Total probability$P(B)=\sum_i P(B|A_i)P(A_i)$Break event B by scenarios/classes $A_i$.
Bayes$P(A_k|B)=\dfrac{P(B|A_k)P(A_k)}{\sum_i P(B|A_i)P(A_i)}$Invert conditional probability after evidence B.
Expectation$E[X]=\sum x_i p_i$ or $\int x f(x)dx$Long-run average.
Variance$Var(X)=E[(X-E[X])^2]=E[X^2]-E[X]^2$Spread around expectation.
Bayes checklist: prior $P(A)$, likelihood $P(B|A)$, evidence $P(B)$, posterior $P(A|B)$.
# Bayes example:
# 2% of users are fraud. Test catches fraud with P(+|fraud)=0.95
# and has false positive rate P(+|not fraud)=0.04.
p_fraud = 0.02
p_not = 1 - p_fraud
p_pos_given_fraud = 0.95
p_pos_given_not = 0.04

p_pos = p_pos_given_fraud * p_fraud + p_pos_given_not * p_not
p_fraud_given_pos = p_pos_given_fraud * p_fraud / p_pos
print(round(p_fraud_given_pos, 4))

Hypothesis Testing, Type I/II Errors, and Correlation

For A-4: know hypothesis testing terminology, error types, statistical power, significance level, and interpretation of the correlation coefficient.

TermMeaningExam wording
$H_0$Null hypothesis, default/no effect.What we try to reject.
$H_1$Alternative hypothesis.Effect/difference exists.
$\alpha$Significance level.Allowed probability of Type I error (typically 0.05).
p-valueProbability of data at least this extreme if $H_0$ is true.If p-value < $\alpha$, reject $H_0$.
Type I error ($\alpha$)Reject true $H_0$. False positive.We declare an effect that does not exist. Controlled by $\alpha$.
Type II error ($\beta$)Fail to reject false $H_0$. False negative.We miss a real effect. Reduced by more data or larger effect.
Power ($1-\beta$)Probability of correctly detecting a real effect.Increases with sample size, effect size, or larger $\alpha$.

Error types summary (decision table):

$H_0$ is true + we reject → Type I error (false positive), rate = $\alpha$.

$H_0$ is false + we fail to reject → Type II error (false negative), rate = $\beta$.

$H_0$ is false + we reject → correct detection, probability = $1-\beta$ (power).

$H_0$ is true + we fail to reject → correct non-rejection, probability = $1-\alpha$.

Trade-off: lowering $\alpha$ reduces Type I errors but increases $\beta$ (more Type II errors).

Pearson correlation: $r=\dfrac{cov(X,Y)}{\sigma_X\sigma_Y}$, ranges from $-1$ to $1$.

$r \approx 1$: strong positive linear relation. $r \approx -1$: strong negative. $r \approx 0$: no linear relation (but nonlinear may exist).

Spearman $\rho$: rank-based; detects any monotonic relationship, robust to outliers.

Linear Models and Gradient Descent

For A-11 and B-2: understand that a linear model computes a weighted sum of features, and gradient descent iteratively minimizes the loss function.

Linear model: $\hat{y}=w_0+w_1x_1+\dots+w_px_p$

Gradient descent update: $w := w-\eta \nabla_w L(w)$

$\eta$ is learning rate. Too small: slow. Too large: divergence or oscillation.

VariantUpdate usesTrade-off
Batch GDAll training data.Stable but slow on large data.
Stochastic GDOne sample.Noisy but fast updates.
Mini-batch GDSmall batch.Standard compromise for neural networks.
import numpy as np

# Manual gradient descent for simple linear regression y = w*x + b
x = np.array([1, 2, 3, 4], dtype=float)
y = np.array([2, 4, 6, 8], dtype=float)
w, b = 0.0, 0.0
lr = 0.01

for _ in range(1000):
    y_pred = w * x + b
    error = y_pred - y
    grad_w = 2 * np.mean(error * x)
    grad_b = 2 * np.mean(error)
    w -= lr * grad_w
    b -= lr * grad_b

print(w, b)

ML Setup & Data-Analysis Pipeline

Formal setup: there is an object space $X$, an answer space $Y$, a training set $\mathcal{D}=\{(x_i,y_i)\}_{i=1}^{\ell}$, and a model $a:X\to Y$. The feature matrix has size $\ell \times d$: rows are objects, columns are features.

Empirical risk / average loss: $Q(a,\mathcal{D})=\dfrac{1}{\ell}\sum_{i=1}^{\ell}L(y_i,a(x_i))$

Regression with MSE: $Q(a,\mathcal{D})=\dfrac{1}{\ell}\sum_{i=1}^{\ell}(a(x_i)-y_i)^2$

Feature engineeringCreating useful features before modelling.Often more important than model choice.

Linear Regression Deep Dive: Features, Metrics, Closed Form

A linear regression model predicts as a weighted sum of features. It is fast, interpretable, and works well on large datasets, but requires proper feature transformations.

Model: $a(x)=w_0+\langle w,x\rangle=w_0+\sum_{j=1}^{d}w_jx_j$

OLS objective: $\dfrac{1}{\ell}\|Xw-y\|_2^2 \to \min_w$

Closed form: $w=(X^TX)^{-1}X^Ty$ if $X^TX$ is invertible.

Ridge closed form: $w=(X^TX+\alpha I)^{-1}X^Ty$.

Feature problemTransformationWhy it helps linear models
Nominal categoryOne-hot encodingEach category gets its own baseline effect.
TextBag of Words / TF-IDFLinear model learns weight for each word/token.
Nonlinear numeric effectBinning / polynomial / interactionsLinear model becomes nonlinear in original variables.
Heavy-tailed feature$\log(x)$ or $\log(1+x)$Compresses very large values.
Periodicity$\sin(x/T)$, $\cos(x/T)$Encodes cycles like day/week/season.

The gradient points in the direction of steepest ascent; the negative gradient points in the direction of steepest descent. Parameters are therefore updated by the rule $w^{(k)}=w^{(k-1)}-\eta_k\nabla Q(w^{(k-1)})$.

Linear Classification, Margins, Logistic Regression, SVM

A linear classifier separates the space with a hyperplane. For binary labels $y\in\{-1,+1\}$ the margin is $M_i=y_i\langle w,x_i\rangle$: if $M_i>0$ the object is correctly classified; the larger $|M_i|$, the higher the confidence.

Linear classifier: $a(x)=\operatorname{sign}(\langle w,x\rangle+w_0)$.

Zero-one loss: $[M_i<0]$ is hard to optimize directly.

Logistic loss: $\log(1+e^{-M})$.

Hinge loss: $\max(0,1-M)$.

Upper bound lossFormulaAssociated idea/model
Logistic$\log(1+e^{-M})$Logistic regression; probability estimation.
Hinge$(1-M)_+$SVM; margin maximization.
Perceptron$(-M)_+$Penalizes only mistakes, no margin band.
Exponential$e^{-M}$AdaBoost-style weighting; very sensitive to noisy negative margins.
Sigmoid$2/(1+e^M)$Smooth bounded surrogate.

Logistic regression

$P(y=1|x)=\sigma(\langle w,x\rangle)=\dfrac{1}{1+e^{-\langle w,x\rangle}}$

$\langle w,x\rangle=\log\dfrac{P(y=+1|x)}{P(y=-1|x)}$ — log-odds.

Log-loss / negative likelihood: $-[y\log p+(1-y)\log(1-p)]$.

Classification, Ranking Metrics, ROC/PR, Lift

Accuracy can be misleading on imbalanced classes: if 95% of objects are negative, an “always negative” classifier already achieves 95% accuracy. Always inspect the confusion matrix, precision, recall, F1, ROC-AUC, PR-AUC, and business metrics.

$precision=\dfrac{TP}{TP+FP}$, $recall=\dfrac{TP}{TP+FN}$, $F1=\dfrac{2PR}{P+R}$.

$FPR=\dfrac{FP}{FP+TN}$, $TPR=\dfrac{TP}{TP+FN}=recall$.

$Gini=2\cdot AUC-1$.

$lift=\dfrac{precision}{(TP+FN)/\ell}$ = precision divided by positive-class prevalence.

F1Harmonic mean of precision and recall.Falls if either precision or recall is low.
Average PrecisionAverage of precision@k over positions of positives.Equivalent to common area approximation under PR curve.
ROC-AUCProbability that a random positive is ranked above a random negative.Can look overly optimistic when positives are extremely rare.
PR-AUCArea under precision-recall curve.More informative for rare positive class.
R-precisionPrecision where number of predicted positives equals number of true positives.Ranking/search style metric.

Threshold rule: increasing classification threshold usually increases precision and decreases recall. Decreasing threshold usually increases recall and decreases precision.

KNN, Kernel/Parzen Weights, Text Distances, Approximate Neighbours

KNN builds no explicit model during training: it memorizes the dataset and for a new object searches for the nearest neighbors. Quality depends almost entirely on the distance function and feature scaling.

Classification: $a(u)=\arg\max_{y\in Y}\sum_{i=1}^{k}[y^{(i)}_u=y]$.

Weighted kNN / Parzen window: $a(u)=\arg\max_y\sum_{i=1}^{k}K\left(\dfrac{\rho(u,x^{(i)}_u)}{h}\right)[y^{(i)}_u=y]$.

Nadaraya-Watson regression: $a(u)=\dfrac{\sum_i K(\rho(u,x_i)/h)y_i}{\sum_i K(\rho(u,x_i)/h)}$.

Decision Tree Theory: Splits, Impurity, Stopping, Missing Values

A decision tree recursively partitions the feature space. An internal node stores a predicate, typically $[x_j<t]$, while a leaf stores the prediction: class probabilities for classification or the mean/median for regression.

Split gain: $Q(R_m,j,t)=H(R_m)-\dfrac{|R_l|}{|R_m|}H(R_l)-\dfrac{|R_r|}{|R_m|}H(R_r)$.

Regression impurity with MSE: variance inside node.

Gini: $G=\sum_k p_k(1-p_k)=1-\sum_kp_k^2$.

Entropy: $H=-\sum_kp_k\log p_k$.

Bagging, Random Forest, Bias-Variance, OOB

Bagging trains many models independently on bootstrap samples and averages their predictions. It is especially useful for unstable high-variance models such as deep decision trees.

Bagging regression/class-score: $a_N(x)=\dfrac1N\sum_{n=1}^{N}b_n(x)$.

Bias-variance idea for squared loss: expected error = noise + bias$^2$ + variance.

If base errors are uncorrelated, averaging can reduce variance roughly by factor $N$; correlation weakens the gain.

Gradient Boosting, XGBoost, Stacking

Boosting builds a composition sequentially: each new model corrects the errors of the current ensemble. For MSE the new model fits residuals $y_i-a_{N-1}(x_i)$; for a general differentiable loss it fits the negative gradients with respect to the predictions.

Additive model: $a_N(x)=a_{N-1}(x)+\eta\gamma_N b_N(x)$.

Pseudo-residual: $s_i^{(N)}=-\left.\dfrac{\partial L(y_i,z)}{\partial z}\right|_{z=a_{N-1}(x_i)}$.

For MSE: $s_i=y_i-a_{N-1}(x_i)$.

For logistic loss: $s_i=\dfrac{y_i}{1+\exp(y_i a_{N-1}(x_i))}$ for $y_i\in\{-1,+1\}$.

Regularization in boostingEffectExam cue
Learning rate $\eta$Shrinks each step.Lower $\eta$ usually needs more trees and can generalize better.
Number of treesMore sequential corrections.Too many can overfit; use validation/early stopping.
Tree depthControls interaction complexity.Boosting often uses shallow trees, around 3-6 levels.
SubsampleTrain each tree on random subset.Stochastic gradient boosting; can reduce overfitting and speed up.
Robust lossLess sensitive to outliers/noisy labels.Exponential loss can overweight noisy negative-margin examples.

XGBoost-style second-order view

XGBoost improves plain gradient boosting by using a second-order Taylor approximation, regularizing tree complexity, and choosing splits based on the gain from optimal leaf values.

Let $s_i=-g_i$ be negative gradient and $h_i$ be second derivative.

Tree regularization: $\gamma J+\dfrac{\lambda}{2}\sum_{j=1}^{J}b_j^2$.

Optimal leaf value in lecture notation: $b_j=\dfrac{S_j}{H_j+\lambda}$ where $S_j=\sum_{i\in R_j}s_i$, $H_j=\sum_{i\in R_j}h_i$.

Split gain: $\dfrac12\left(\dfrac{S_L^2}{H_L+\lambda}+\dfrac{S_R^2}{H_R+\lambda}-\dfrac{S^2}{H+\lambda}\right)-\gamma$.

Logistic Regression

Linear classifier using sigmoid (binary) or softmax (multiclass). Despite the name, it's a classification algorithm. It estimates class probabilities; calibration is often reasonable for well-specified data but not guaranteed.

Binary: $P(y=1|x) = \sigma(w \cdot x + b) = \dfrac{1}{1+e^{-(w \cdot x + b)}}$

Loss: $-\dfrac{1}{n}\sum_i \left[y_i \log \hat{p}_i + (1-y_i)\log(1-\hat{p}_i)\right]$

max_iterMax iterations for solver convergence (increase if ConvergenceWarning)
multiclasssklearn 1.8 uses multinomial loss for multiclass when supported; use OneVsRestClassifier if one-vs-rest is required.

Transforms each feature to zero mean and unit standard deviation. Required by many algorithms (SVM, logistic regression, KNN, PCA). Always fit on train set only.

$z = \dfrac{x - \mu}{\sigma}$  where $\mu$ = feature mean, $\sigma$ = feature std dev

Normalisation

Two different concepts — make sure you pick the right one:

MinMaxScaler (feature scaling): $x' = \dfrac{x - x_{\min}}{x_{\max} - x_{\min}}$ → maps to $[0, 1]$

Normalizer (sample normalisation): scales each row to unit $\ell^p$ norm: $x' = \dfrac{x}{\|x\|_p}$

from sklearn.preprocessing import MinMaxScaler, MaxAbsScaler, Normalizer

X_mms_test = mms.transform(X_test)

# ── MaxAbsScaler: scale by max abs value → [-1,1] ──
# Good for sparse data (doesn't shift mean)
mas = MaxAbsScaler()
X_mas = mas.fit_transform(X_train)

# ── Normalizer: L2-normalise each SAMPLE (row) ──
# Each row has ||x||₂ = 1 after transformation
norm = Normalizer(norm='l2')  # 'l1' or 'max' also available
X_norm = norm.fit_transform(X_train)

from sklearn.pipeline import Pipeline

    ('cat', categorical_pipe, cat_cols)

    ('prep', preprocess),

])

)

}

gs.fit(X_train, y_train)

y_pred = gs.predict(X_test)
print(gs.best_params_)
print(f1_score(y_test, y_pred, average='macro'))
Scikit-learn metrics

Classification Metrics

Notation: TP = true positives, TN = true negatives, FP = false positives, FN = false negatives.

Classification

Accuracy

$\dfrac{TP+TN}{TP+TN+FP+FN}$

Proportion of correct predictions. Misleading on imbalanced datasets.

accuracy_score(y_true, y_pred)
Classification

Precision

$\dfrac{TP}{TP+FP}$

Of all predicted positives, how many are actually positive. High precision = few false alarms.

precision_score(y_true, y_pred)
Classification

Recall (Sensitivity)

$\dfrac{TP}{TP+FN}$

Of all actual positives, how many did we catch. High recall = few missed positives.

recall_score(y_true, y_pred)
Classification

F1-Score

$\dfrac{2 \cdot P \cdot R}{P + R} = \dfrac{2\,TP}{2\,TP+FP+FN}$

Harmonic mean of precision and recall. Good single metric for imbalanced data.

f1_score(y_true, y_pred)
Classification

ROC-AUC

Area under TPR vs FPR curve.
$TPR = \tfrac{TP}{TP+FN}$, $FPR = \tfrac{FP}{FP+TN}$

1.0 = perfect, 0.5 = random. Requires probability scores, not class labels.

roc_auc_score(y_true, y_score)

Regression Metrics

Regression

MAE — Mean Absolute Error

$\dfrac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i|$

Robust to outliers. Same unit as target. Easy to interpret.

mean_absolute_error(y_true, y_pred)
Regression

MSE — Mean Squared Error

$\dfrac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$

Penalises large errors heavily. Unit is target². Take sqrt for RMSE.

mean_squared_error(y_true, y_pred)
Regression

R² — Coefficient of Determination

$1 - \dfrac{SS_{res}}{SS_{tot}} = 1 - \dfrac{\sum(y_i-\hat{y}_i)^2}{\sum(y_i-\bar{y})^2}$

1.0 = perfect fit; 0.0 = predicts mean; negative = worse than predicting mean.

r2_score(y_true, y_pred)

Clustering Metrics

For clustering metrics that require ground-truth labels: use when you have labelled data to evaluate cluster quality. Silhouette and Davies-Bouldin work without labels.

Clustering

Adjusted Rand Score

$ARI = \dfrac{RI - E[RI]}{\max(RI) - E[RI]}$

Adjusted for chance: 1.0 = identical, 0.0 = random, can be negative. Symmetric.

adjusted_rand_score(labels_true, labels_pred)
Clustering

Homogeneity

Each cluster contains only members of one class.
$h = 1 - H(C|K)/H(C)$

Range [0,1]. High = clusters are "pure".

homogeneity_score(labels_true, labels_pred)
Clustering

V-Measure

$V_\beta = \dfrac{(1+\beta) \cdot h \cdot c}{\beta \cdot h + c}$

Harmonic mean of homogeneity (h) and completeness (c). β weights completeness.
